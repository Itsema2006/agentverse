1. Overview

Welcome to the AgentVerse Developer Hub. The AgentVerse API allows you to programmatically trigger, chain, and orchestrate production-ready autonomous AI agents. All operations are authenticated using secure API keys that map to specific agent services. 

Base API URL: https://api.agentverse.ai/v1

2. API Key Setup Flow

To use the AgentVerse developer platform, you must first obtain a key for the agent you wish to run. Follow the setup lifecycle below:

1

Browse Agents

Navigate to the Explore page to browse available AI agents and select the key option that fits your needs.

2

Purchase Key

Click "Buy Key" on the browse card to open the secure payment checkout. Complete the payment to unlock the key.

3

Copy from Dashboard

Navigate to your User Dashboard. Scroll to the "Owned API Keys" section to copy your active key value.

4

Use in Code

Securely add your key to your project's local environment settings to start making authenticated calls.

3. Key Installation

API keys must never be exposed in public source control repos. Install your key by setting it as an environment variable in a local .env file in the root of your application: 

# .env file
AGENTVERSE_API_KEY=av_key_your_purchased_key_here

Make sure to add your .env file to your project's .gitignore rules:

# .gitignore
.env

4. SDK Integrations

AgentVerse provides official, type-safe SDKs to quickly run autonomous workflows.

Node.js Integration

Install the package:

npm install @agentverse/sdk dotenv

Initialize and run task:

import dotenv from 'dotenv';
import { AgentVerse } from '@agentverse/sdk';

dotenv.config();

const client = new AgentVerse({
  apiKey: process.env.AGENTVERSE_API_KEY
});

// Run an agent task
const response = await client.agents.run({
  agent: 'CodeGenPro',
  task: 'Create a secure JWT authentication middleware in Node.js',
  temperature: 0.2
});

console.log('Result:', response.output);

Python Integration

Install the library:

pip install agentverse-sdk python-dotenv

Initialize and run task:

import os
from dotenv import load_dotenv
from agentverse import AgentVerse

load_dotenv()

client = AgentVerse(
    api_key=os.getenv("AGENTVERSE_API_KEY")
)

# Run an agent task
response = client.agents.run(
    agent="CodeGenPro",
    task="Create a secure JWT authentication middleware in Python",
    temperature=0.2
)

print("Result:", response["output"])

5. Core Endpoints

If you prefer to make raw HTTP requests, you can construct an authorization header as follows:

Authorization: Bearer <YOUR_API_KEY>
Method Path Description
POST /agents/run Execute a task on an agent.
GET /agents/list List all available agent options.

Example raw cURL request:

curl -X POST https://api.agentverse.ai/v1/agents/run \
  -H "Authorization: Bearer $AGENTVERSE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "agent": "CodeGenPro",
    "task": "Create a secure auth middleware",
    "temperature": 0.2
  }'

6. Error Handling

Handle API status codes in your application integration with standard fallback policies:

  • 400 Bad Request: Invalid payload parameters. Validate request parameters.
  • 401 Unauthorized: Key invalid, inactive, or unauthorized. Check your Dashboard key value.
  • 429 Too Many Requests: Rate limit exceeded. Implement exponential backoff in client requests.
  • 500 Server Error: Internal agent failure. Capture transaction ID and retry after interval.
