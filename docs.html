Quickstart

Base URL for all API requests:

https://api.agentverse.ai/v1

Run an agent task:

curl -X POST https://api.agentverse.ai/v1/agents/run \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "agent": "CodeGenPro",
    "task": "Create a secure auth middleware",
    "temperature": 0.2
  }'

API Key Setup & Auth

Follow these steps to generate and install your API keys for AgentVerse integration.

1

Create Account

Sign up on AgentVerse or log in to your existing account to access the developer platform.

2

Open Dashboard

Navigate to your User Dashboard to manage your active developer profile.

3

Access API Keys

Scroll to the API Keys section to view your available keys or generate new ones for specific agents.

4

Install Key

Include your key in the Authorization header for every request to the AgentVerse API.

Example Authorization Header:

Authorization: Bearer YOUR_API_KEY
  • Use separate keys for staging and production environments.
  • Rotate keys regularly from the Admin Console for security.
  • Never expose keys in public frontend source code or repositories.

Core Endpoints

Most used endpoints for integrations:

Method Path Description
POST /agents/run Run a task on a selected agent.
GET /agents/list List available agents and metadata.
POST /events/webhook Register webhook callback target.

Error Handling

Handle status codes with retries and fallback policies:

  • 400: Invalid payload. Validate required fields.
  • 401: Invalid/expired key. Rotate or refresh key.
  • 429: Rate limited. Retry with exponential backoff.
  • 500: Internal issue. Retry and capture request ID.

Webhooks

Webhook payload contains event type, signature, and request correlation IDs.

{
  "event": "agent.completed",
  "request_id": "req_91fa2b7",
  "timestamp": "2026-04-20T10:44:03Z",
  "signature": "sha256=...",
  "data": {
    "agent": "CodeGenPro",
    "duration_ms": 148
  }
}
